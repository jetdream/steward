FINDINGS:
- [severity: high] CTX-2 v1 normative text is violated by its own shipped implementation and is internally contradictory — the rule's first sentence scans only "ID tokens with registered prefixes", yet its last sentence demands "unresolved **or unregistered** tokens are noted rather than silently skipped"; verified empirically: `scripts/hooks/resolve-ids-hook.mjs:25` filters unregistered prefixes out silently (prompt containing `UTF-8`, `ZZZZ-9`, `MEM-1`, `MEM-999` noted only `MEM-999`). The implementation is the defensible side: the ID grammar collides with everyday tokens (`UTF-8`, `SHA-256`, `ISO-9001`), and noting them per-prompt would defeat CTX-3's anti-pollution mandate — so the spec text, if implemented literally, would be harmful. An approved spec whose normative rule its own module contradicts cannot stand. Fix: bump CTX-2 to v2, scoping the "noted" clause to registered-prefix tokens that fail to resolve (current behavior), and state explicitly that unregistered-prefix tokens are ignored to avoid grammar collisions.
- [severity: high] Status `approved` is currently unlawful — the spec carries no `challenge:` block, violating DCX-13 v1 of its own `depends-on` [DCX]; `node scripts/docs-check.mjs` is red right now with exactly this error (`.spec/specs/ctx-context-hooks.yaml:1 DCX-13: approved spec needs an Architect Challenger record`). Survives scrutiny because the gate is lint-enforced and failing on main, and DCX-13 forbids recording a verdict that was never produced. Fix: regress status to `draft`, resolve the CTX-2 finding above, re-run the challenger, and record the resulting verdict in a `challenge:` block before re-approving.
- [severity: medium, non-blocking on its own] Purpose overclaims the invariant: "no change can be written against a contract the session never loaded" — but CTX-5 guards only `Edit|Write` tool calls; file writes via Bash (`sed -i`, heredocs, `git apply`), NotebookEdit, or any future write-capable tool bypass the guard entirely. The `edge-cases` section honestly lists the other known limits (partial Read, compaction) but omits this one, making it a hidden assumption rather than an accepted limit. Fix: soften the purpose claim to the Edit/Write channel and add the bypass channels to `edge-cases` as an accepted v1 limit.
- [severity: low, non-blocking] Ledger key fallback is unspecified and leaks across sessions: all three ledger-using hooks fall back to a shared `steward-ctx-default.json` when `session_id` is absent (`scripts/lib/session-ledger.mjs:14`, verified by invocation without `session_id`), while the spec's `data` section defines only `steward-ctx-<session_id>.json`. Doesn't block: the harness always supplies `session_id`, and the worst case is one lost deny-once roundtrip — but the fallback should either be specified or removed (fail open on missing id).
- [severity: low, non-blocking] "Mentioned but not defined anywhere" notes (CTX-2) are never marked seen, so an undefined-but-registered token is re-noted on every prompt that mentions it — a small leak in CTX-3's "never become context pollution". Doesn't block: repetition is bounded to one line per prompt and arguably a useful reminder; decide intentionally and record it in `defaults-and-assumptions`.
