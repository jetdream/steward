# backend/src/ — the `@backend` module root & API seam

The capability modules each live in their own folder (see their `CLAUDE.md`s):
`memory`, `content`, `topics`, `onboarding`, `accounts`, plus the AI substrate
(`harness`, `observability`, `ports`, `adapters`, `eval`), identity (`auth`), and
persistence (`db`). This top level holds the **API seam** that composes them:

| File | Role |
|---|---|
| `server.ts` | Dev entrypoint — HTTP (tRPC) + WebSocket + `/api/auth/*` (BetterAuth) on one port; imports `otel.ts` first |
| `trpc.ts` | tRPC init + `publicProcedure` / `protectedProcedure` / `orgProcedure` (the ACCS-2 confinement seam) |
| `context.ts` | Per-request/-socket `Context` (db, auth, memory, session, cookie append) |
| `router.ts` | `appRouter` — composes the per-capability routers; exports the `AppRouter` client contract |
| `otel.ts` | OpenTelemetry bootstrap |

Routers are thin (`routers/CLAUDE.md`); business logic lives in the capability
modules. The client calls only through these routers (constitution "Client").
