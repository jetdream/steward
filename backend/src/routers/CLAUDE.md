# backend/src/routers/ — tRPC capability routers

**Purpose.** One router file per capability, composed into the app router by the
thin composition root (`../router.ts`). Each is the domain API surface the
client calls (constitution "Client" — never the backend directly). Routers stay
thin: they validate input (Zod), enforce org confinement via `orgProcedure`
(ACC-3), and delegate to the capability module (`../memory`, `../onboarding`,
`../auth`); business logic never lives here.

| Router | Surface |
|---|---|
| `auth.ts` | dev-login / me / logout (ACCS-1) |
| `org.ts` | org list/create/active + membership lifecycle (ACCS-1/2/3) |
| `memory.ts` | `remember` / `context` / `shouldAsk` over `@backend/memory` (MEMS-1/4/5) |
| `onboarding.ts` | `gaps` / `profile` / `correct` / `ready` over `@backend/onboarding` (ONBS-3/5/6) |
| `system.ts` | infra health probes (pgvector round-trip) |
| `ping.ts` | WS subscription demo |
