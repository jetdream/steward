# backend/src/strategy/ ‚Äî the Posting Strategy (ARC-14 / STR)

**Purpose.** Produces the StrategyDoc (DM-3) that grounds generation ‚Äî the
five-section editorial contract (STR-1). Implements the behaviour spec **STRS**
(`.spec/specs/str-posting-strategy.yaml`).

| File | Role | Spec |
|---|---|---|
| `store.ts` | DM-3 append-only version store (`getCurrentDoc` / `insertVersion` / `listVersions`) + the Memory reads the view needs (`activeGroundingText`, `activeOverlay`) | `@implements STRS-1/STRS-2` |
| `index.ts` | `createStrategy` facade + `getStrategy` (5-section view, c derived), `autoDraft` (the `draft-strategy` Skill), `editSection` + `routesToMemory` (STRS-2 semantic routing) | `@implements STRS-1/2/4` |

**The storage split (DEC-22).** Sections (a/b/d/e) PERSIST and version (STR-2 ‚Äî
each edit appends a row; current = max version; prior retained, VAL-3). Section
(c) is a live DERIVED view assembled by `getStrategy`: the **platform** layer
(`PLATFORM_GUARDRAILS`, GR-1..GR-6 read-only) + the **org** layer (the active
Memory rule/taboo overlay, `activeOverlay`). It is NEVER stored here.

**Semantic edit routing (STRS-2, `routesToMemory`).** A section (c) edit ‚Äî or a
hard PROHIBITION typed in section (a) ‚Äî is written to **Memory** as a rule/taboo
(binds via the (c) overlay, MEMS-3), reusing the SAME MEMS-1 correction-channel
classification (`applyCorrectionPolicy`), never a new heuristic (LRN-20). Every
other edit is a soft editorial change ‚Üí a new StrategyDoc version. A prohibition
can never earn soft-only enforcement by placement. Both bind immediately (STRS-2).

**Auto-draft (STRS-2).** `autoDraft` grounds the `draft-strategy` Skill (ARC-27)
in the org's active Memory and persists sections (a/b/d/e) as v1 ‚Äî never a blank
page (VAL-6), nothing invented (VAL-4).

**Deferred.** STRS-3 enforcement (`validateDraft` at the PIPE-2 VAL stage) is
consumed by the GEN pipeline ‚Äî the soft strategy-fit + the hard channel-fit gate
(GENS-5) land with GEN. Section (e) targets the launch channels until
connected-channel filtering lands with ChannelConnection (ONBS-4).

**Boundaries.** Org-scoped everywhere (ACC-3). Memory is the single source
(DEC-22) ‚Äî reads go through `memory.write` for edits + org-scoped queries for the
derived overlay/grounding (the same direct-read pattern onboarding uses).
