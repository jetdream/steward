# backend/src/accounts/ — accounts/identity policy (ARC-10)

**Purpose.** The Steward-specific POLICY over BetterAuth identity (ADR-0006,
SEC-7). Identity itself — users, sessions, org-plugin memberships — is
BetterAuth's; this module owns only what the plugin does NOT enforce. Today that
is the **single-owner invariant** (ACCS-3): the org plugin treats `owner` as an
ordinary role a role-change would freely duplicate, so Steward guarantees exactly
one owner per org here.

| File | Role | Spec |
|---|---|---|
| `ownership.ts` | Pure policy: `guardRoleChange` / `guardMemberRemoval` (the owner is untouchable through those paths) + `planOwnershipTransfer` (the atomic demote-owner/promote-target swap) + `OwnershipError` | `@implements ACCS-3 v1` |
| `index.ts` | Barrel — the surface the org router consumes | — |

**Consumer.** `routers/org.ts` is the thin caller: it loads the member rows,
asks this module for the decision, applies the DB writes (the transfer in one
transaction), and maps `OwnershipError.code` onto the matching `TRPCError` code.
This module is **transport-free** (no tRPC import) and **deterministic** — no LLM,
no content judgment (LRN-20).

**Boundaries.** The BetterAuth mechanism (plugins, sign-in, session) lives in
`../auth/`; the org/session tRPC surface is `../routers/{auth,org}.ts`;
server-side org confinement (ACCS-2) is `orgProcedure` in `../trpc.ts`.
